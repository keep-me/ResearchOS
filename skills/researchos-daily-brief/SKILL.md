# ResearchOS Daily Brief

优先复用系统内置简报能力，不手工拼接长列表。

推荐流程：

1. 确认用户是否指定了接收对象、订阅范围或研究方向。
2. 调用 `generate_daily_brief` 生成简报。
3. 对生成结果做简短整理，突出最值得关注的更新。

输出时优先强调：

- 今日重点论文或方向变化
- 为什么值得看
- 建议下一步阅读顺序

如果当前配置不足以生成个性化简报，要明确说明缺少哪些条件。
